Pre-launch statement: This page records the target and current work. It must be updated after formal keyboard, screen-reader, contrast, zoom, reduced-motion, and touch testing.

Target

We are preparing CrewMultiply Play toward WCAG 2.2 Level AA for the website shell and toward equivalent practical access in each game where the visual mechanic allows it.

Website support

  • Keyboard-accessible navigation and visible focus.
  • Semantic headings, links, buttons, and landmarks.
  • Text alternatives for meaningful screenshots.
  • Responsive layout through phone widths and browser zoom.
  • Reduced-motion support based on device preference.
  • No information conveyed by color alone.

Game support plan

  • Pointer, touch, and keyboard actions through one shared control layer.
  • Large touch targets and controls positioned outside active puzzle cells.
  • High-contrast and color-symbol modes.
  • Independent controls for music, effects, motion, and camera impulse.
  • Pause on focus loss and no penalty while menus are open.
  • Rule text, state summaries, and non-timed modes where the mechanic permits.

Known work before launch

The existing game prototypes need a full keyboard and assistive-technology pass. Some canvas-style puzzles may require parallel state descriptions or alternate controls. Each game page will publish its specific limitations rather than relying on one broad claim.

Accessibility feedback

Planned contact: accessibility@crewmultiply.com. Activate this mailbox before publishing and include a response-time goal.
